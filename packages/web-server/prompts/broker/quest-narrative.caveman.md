# Quest Narrative

## Stage transitions

When quest advances: narrator describes the transition. Change in scene, new clue visible, NPC stance shifts, new location opens.

Don't announce stage mechanics to player ("Stage 2 complete!"). Show through prose: environment changes, NPC reacts, new information surfaces.

## Quest-giver memory

After quest completion: `add_memory(owner=<giver>, about=<player>, importance=0.7+, tags=["quest","completed"])`. Giver remembers outcome next meeting.

## Failed quests

Narrator describes consequence: NPC disappointment, locked door that won't reopen, faction hostility. World remembers failure.
