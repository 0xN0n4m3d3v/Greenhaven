# @Entity Name

Коротко: одна-две фразы, зачем эта сущность нужна в мире и что игрок
почувствует при встрече с ней.

## Канон

- Тип: локация / сцена / персонаж / предмет / квест / действие-открытие
- Где находится: @Parent location
- Видимость: видно сразу / скрыто до действия / слух / только для агента
- Главная функция в игре: торговля / разговор / путь / угроза / награда / тайна

## Как это выглядит

Опиши обычным языком. Можно писать как для другого мастера: запахи, свет,
звуки, одежда, состояние предмета, настроение сцены.

## Как это работает

Что игрок может сделать и что меняется в мире.

Примеры:

- если герой сдвинет @Barrels in the square, откроется путь в @Thief's market;
- если герой поможет @Mikka, она получает +1 string доверия к герою;
- если герой заберет предмет, он должен появиться в инвентаре.

## Materializes

Если эта сущность ничего не создает и не открывает, этот блок можно удалить.

- Когда происходит ...:
  - Entity: @New Entity Name
  - Type: location / item / NPC / scene / quest / service / access / state
  - Scope: @Parent location
  - Effect: ...

## Связи

- Находится в: @Parent location
- Ведет к: @Target location
- Связано с: @NPC or item
- Запускает квест: Quest Name

## Картинки

Картинки клади рядом с этой заметкой в `images/`, `portraits/` или `media/`.
В тексте можно описать, какая картинка нужна.
